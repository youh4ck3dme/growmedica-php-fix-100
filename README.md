## 🚀 Quick Start

### Prerequisites
- Node.js 18+ (tested on 22)
- Yarn 1.22+

### Local Development
```bash
cd storefront

# Install dependencies
yarn install

# Pull environment variables (development)
yarn pull:env

# Start dev server (Turbopack)
yarn dev

# Open: http://localhost:5555
```

### Build & Deploy
```bash
# Type check
yarn type-check

# Lint
yarn lint

# Build production
yarn build

# Start production server
yarn start

# Open: http://localhost:3000
```

### Testing
```bash
# Integrity tests (67 tests; Shopify API mocked for Playwright webServer)
yarn test:integrity

# PWA validation (offline, manifest, SW)
yarn test:pwa

# E2E tests (product pages, collections)
yarn test:e2e

# All tests with UI
yarn test:ui
```

## 🔧 Configuration

### Environment Variables
**.env.local** (create locally, not in git):
```bash
SHOPIFY_STORE_DOMAIN=your-store.myshopify.com
SHOPIFY_STOREFRONT_ACCESS_TOKEN=your-storefront-access-token
SHOPIFY_REVALIDATION_SECRET=your-random-webhook-secret-min-16-chars
SHOPIFY_API_VERSION=2025-01
NEXT_PUBLIC_SITE_URL=http://localhost:5555
```

Legacy `NEXT_PUBLIC_SHOPIFY_*` names are still read during migration, but prefer the
server-side `SHOPIFY_*` variables above. The Storefront token is not an Admin
`shpat_*` token.

Pull from Vercel development env:
```bash
yarn pull:env
```

### PWA Manifest
**public/manifest.webmanifest** — Defines:
- App name, description, icons
- Home screen theme color, background color
- Start URL, scope, display mode
- Categories for app stores

See [PWA Setup Guide](./storefront/docs/PWA_SETUP.md) for details.

### Service Worker (Serwist)
**src/app/sw.ts** — Caching strategy:
- **Network-first** for HTML routes (offline fallback → `/offline`)
- **Cache-first** for CSS, JS, fonts, images
- **NetworkOnly** for API calls (5s timeout)

## 🌐 PWA Features

### Installable
- Works on Chrome, Edge, Android
- Install prompt: "Inštalujte GrowMedica.sk" banner
- Add to home screen (iOS: manual via "Share" menu)

### Offline Support
- Cached pages available offline
- Offline fallback page: `/offline` or `/offline.html`
- API calls timeout after 5s → graceful fallback

### Install Banner
Component: `src/components/layout/PwaInstallBanner.tsx`
- Triggers on `beforeinstallprompt` event (Chrome, Edge)
- Dismissible (localStorage)
- Not shown on iOS Safari (app install via home screen)

See [PWA Setup Guide](./storefront/docs/PWA_SETUP.md) for testing & troubleshooting.
